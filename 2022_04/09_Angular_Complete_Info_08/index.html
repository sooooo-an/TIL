1. About Forms

Angular는 SPA이므로 Form을 서버로 전송시킬 필요는 없지만, 만약 서버로 무언가를 전송시키기 위해서는 Angular에서 제공해주는 HTTP 서비스를 이용해야한다. 그렇다면 우리는 왜 Angular의 도움이 필요할까?

<form>
  <label>Name</label>
  <input type="text" name="name" />
  <label>Mail</label>
  <input type="text" name="email" />
  <button type="text">Save</button>
</form>


<!--
{
    value: {name: '', email: ''}
}
-->
  • HTML 코드만 사용한다면, 스스로 key-value pairs의 형식(데이터를 객체로 맵핑하는게 작업하기에 쉽기 때문에)으로 Form의 데이터를 맵핑해주어야하기 때문에 TypeScript를 작성하는데 힘들다. 그렇기 때문에 Angular에서 제공해주는 내장 기능을 사용한다면,Form 데이터를 key-value pairs의 형식으로 표현해주며, 메일주소가 유효한 메일 주소인지 확인하는 등의 작업 또한 Angular에서 작업할 수 있기 때문에 Angular를 사용하는게 편리하다.

2. Template-Driven(TD) vs Reactive Approach

Template-Driven

  • Angular가 DOM에서 Form 객체를 추론한다

  • Form을 설정하는데 쉽게 설정할 수 있다 → Form에 어떤 컨트롤이 존재하는지, 어떤 입력이 있는지 추론하고 빠르게 시작할 수 있다

  • Reactive

    • Form은 프로그래밍 방식으로 생성되고 DOM과 동기화된다.
    • Form을 설정하는데 복잡하다 → TypeScript 파일에 구조를 정의해야하고, HTML 코드도 설정한 뒤 수동으로 연결해야한다.

3. TD: Create Form & Registering Form Controls

  • FormModule을 app.module.ts에 import해주는 것이 중요하다. 우리는 Angular가 Form을 자동으로 감지할거라고 생각하지만, Angular는 자동으로 감지하지 못한다 → WHY?

    • 우리가 사용하는 모든 폼을 컨트롤하고 싶지 않을 수도 있기 때문에
    • Input 레이블이 지정되지 않은 일부 커스텀 폼 컨트롤을 추가하고 싶을 수도 있기 때문에 이런 부분을 통제할 수 있도록 컨트롤을 수동으로 등록해야한다.

  • 그렇다면 Angular가 실제로 제어하는 것은 무엇일까?

  <input type="text" id="username" class="form-control" ngModel />
  • ngModel을 작성하면, Input 엘리먼츠를 제어한다는 뜻으로 ngModel은 폼 모듈에서 사용할 수 있는 디렉티브이다. (ngModel → two-way binding 방법)
  • Angular는 Form을 제어하기 위해서 이에 대한 정보가 필요하다. 사용자가 작성하는 name 이라는 프로퍼티를 통해서 Form에 대한 정보를 알려줄 수 있다.
  <input type="text" id="username" class="form-control" ngModel name="username"/>

4. TD: Submit & Using Form

엔터를 클릭하면 유저가 전송을 할 수 있도록 만들자. 자바스크립트의 submit 이벤트를 사용하는 것보다는 Angular의 내장 디렉티브인 ngSubmit을 사용하는 것이 더 좋다. 자바스크립트의 submit 이벤트를 사용하게 된다면, Form을 submit하고 정상적으로 동작을 보내지만, JavaScript에 내장된 submit 이벤트 외에도 HTML에 내장된 JavaScript 이벤트도 트리거하기 때문이다.

  • ngSubmit: 이 디렉티브는 Form이 Submit될 때마다 트리거된다.
<form (ngSubmit)="onSubmit(form)" #form></form>
export class InputComponent {
  // typescript code
  onSubmit(form: HTMLFormElement) {
    // form object
  }
}
  • 위의 코드대로 Submit 버튼을 클릭하게 되면, form object가 파라미터 값으로 들어오는 것을 알 수 있으나, 이는 우리가 원하는 form object가 아니다.
  • 해결방법 → 타입을 NgForm으로 변경해준다.
<form (ngSubmit)="onSubmit(form)" #form="ngForm"></form>
export class InputComponent {
    // typescript code
    onSubmit(form: NgForm) {
        // form object
    }
}
  • 이 방식은, Angular가 자동으로 생성한 Form에 대한 액세스를 부여해주는 방법이다. HTML에서 form을 넘김으로써 더 이상 인자의 타입이 ElementRef가 아닌, NgForm이 된다.

5. TD: Form State 이해하기

  • control: 각 컨트롤은 FormControl 유형.
  • dirty: 변경이 있음을 체크
  • touch: Field 중 일부를 클릭했는지 확인 → dirty는 필드 및 필드 변경 / touch는 클릭
  • disabled, invalid, value

6. TD: Form에서 @ViewChild 접근하기

템플릿(HTML)에서 로컬 레퍼런스를 통해 Form을 보내는 것이 아닌 타입스크립트 코드에서 viewChild를 이용해서 접근할 수 있도록 변경하기

export class InputComponent {
    @VideChild('f') signupForm: NgForm;


    onSubmit() {
        console.log(this.signupForm);
    }
}

7. TD: Validation 추가하기

<input type="text"
             id="username"
             class="form-control"
             ngModel
             name="username"
             required/>


<input type="email"
             id="email"
             class="form-control"
             ngModel
             name="email"
             required
             email/>
  • required: 기본적인 HTML Attribute. 그러나 여기에서는 Angular가 required를 감지하여 Angular와 함께 제공되는 내장 디렉티브로 selector 역할을 한다. 이 Form이 유효한지에 대한 부분을 확인할 수 있는 디렉티브이다.
  • email: 기본적인 HTML Attribute는 아니지만, Angular와 함께 제공되는 내장 디렉티브로, selector 역할을 한다. email은 유효한 email인지 확인할 수 있는 디렉티브이다.
  • 위의 내장 디렉티브들을 사용했는데도 불구하고, 폼이 빈값이여도, 이메일 형식으로 작성하지 않아도 submit 버튼을 동작한다. 그러나 이 폼에 대한 내용을 로그에 출력해보면, valid라는 프로퍼티가 false인 것을 확인할 수 있다.
    • Angular의 폼은 state를 트래킹하고, 폼이 유효한지에 대해서 알려준다. 이는 폼을 조금 더 편하게 사용할 수 있도록 도와준다.
    • 이 뿐만 아니라, email 엘리먼츠를 확인하면, ng-dirty, ng-touched, ng-valid 라는 form-control 클래스들이 있는 것을 확인할 수 있다. 여기서 이메일 인풋에서 @을 제거하게 되면, ng-valid라는 클래스가 지워지는 것을 볼 수 있다. 이는 Angular가 동적으로 클래스를 추가했기 때문이다.
    • Angular는 Input이 변경되었는지 아닌지(ng-dirty), 클릭을 했는지 안했는지(ng-touch), 유효성 검사를 통과했는지 아닌지(ng-valid)에 따라서 동적으로 클래스를 추가해준다.

9. TD: Form State 사용하기

유효성 검사에 실패하더라도 현재는 Submit 버튼을 클릭할 수 있도록 되어있다. 유효성 검사에 실패한다면, 버튼을 클릭할 수 없도록 만들어보자.

<form #f="ngForm">
  <button
    class="btn btn-primary"
    type="submit"
    [disabled]="!f.valid"></button>
</form>
.ng-invalid {
    border: 1px solid red;
}
  • 위의 방식으로 만들게되면 각 에러가 있는 폼에만 보더로 감싸지는 게 아닌, 폼전체가 보더로 감싸지게 된다. Angular는 input 뿐만 아니라 form 엘리먼츠에도 ng-invalid 클래스를 추가하기 때문이다.
  • 위의 부분을 해결하는 방법은 여러가지가 있는데, 그중 한 가지 방법은 pure CSS 로직이다.
    • input에만 boder를 추가하고 싶으므로, input.ng-invalid, select.ng-invalid를 이용해서 추가할 수 있다. → 그러나 이 방법에도 1가지 이슈가 있다. Form의 초기값은 빈값이므로, input과 select는 ng-invalid 클래스를 가지고 있다. 즉 이 말은, 사용자가 값을 작성하기도 전에 에러 보더가 나오게 된다는 뜻이다.
    • 위의 이슈를 해결하기 위해서는, 우리는 ng-touched라는 클래스를 이용할 수 있다. ng-touched는 1번의 클릭 후에 추가되는 클래스로, 맨 처음에 에러 보더가 나오는 것을 방지할 수 있다. → input.ng-invalid.ng-touched

10. TD: Validation Error Message 방출하기

<input
    type="email"
    id="email"
    class="form-control"
    ngModel
    name="email"
    required
    email
    #email="ngModel"/>
<span class="help-block"
            *ngIf="!email.valid">Please enter a valid email!</span>
  • input 엘리먼츠에 로컬 레퍼런스를 추가하여 Angular가 제공해주는 컨트롤에 쉽게 접근할 수 있다. #email=”ngModel”을 사용하면, #f=”ngForm” 과 마찬가지로 ngModel 디렉티브로 생성되는 컨트롤에 대한 정보를 얻을 수 있다.
  • *ngIf=”!email.valid”: 이메일이 유효성검사에 통과하지 않을 때만 화면에 출력된다. (email.touched 또한 추가하면, 맨 처음에는 span 엘리먼츠가 화면에 출력되지 않는다.)

11. TD: Set Default Value with ngModel Property Binding

<select
    id="secret"
    class="form-control"
    [ngModel]="'pet'"
    name="secret">
    <option value="pet">Your first Pet?</option>
    <option value="teacher">Your first Teacher?</option>
</select>
  • ngModel을 대괄호만 사용하여 값을 설정하게되면, default value를 설정할 수 있게된다.
  • ngModel은 바인딩 없이 사용할수도 있고, one binding, two binding, default value 등 다양하게 활용하여 사용할 수 있다.

12. TD: Using ngModel with Two-Way Binding

ngModel을 사용할 때, 굳이 two-way 바인딩을 사용하지 않아도 된다. ngModel은 어떤 바인딩도 없이 또는 one-way 바인딩을 통해서 사용할 수 있기 때문이다. 그러나 사용자가 입력한 내용을 “즉시" 확인하고 싶을 때는 two-way 바인딩을 사용할 수 있다.

<div class="form-group">
    <textarea name="questionAnswer"
                      row="3"
                        [(ngModel)]="answer"></textarea>
    <p>Your reply: {{answer}}</p>
</div>
  • 유저가 작성한 답을 유저에게 즉각적으로 보여주기 위해서, two-way 바인딩을 사용하여 이를 가능하게 할 수 있다.
  • name=”questionAnswer”: 폼을 Submit할 때, textarea의 값은 questionAnswer의 프로퍼티에 담긴다.

13. TD: Form Control 그룹핑하기

화면에 컨트롤해야 하는 폼이 1개가 아니라 여러개일 경우, 각 폼을 그룹으로 만들어서 사용하는 방법도 있다.

<div id="user-data" ngModelGroup="userData" #userData="ngModelGroup">
    <div class="form-group">
        <label for="username">Username</label>
        <input
            type="text"
            id="username"
          class="form-control"
          ngModel
            name="username"
            required />
    </div>
    <button
        class="btn btn-default"
        type="button">Suggest an Username</button>


    <div class="form-group">
        <label for="email">Mail</label>
        <input
            type="email"
            id="email"
            class="form-control"
            ngModel
            name="email"
            required
            email
            #email="ngModel"/>
    </div>
</div>
<p *ngIf="!userData.valid && userData.touch">User Data is invalid!</p>
  • ngModelGroup 디렉티브를 추가한다. ngModelGroup은 스트링인 그룹키를 값으로 한다 (userData)
  • 버튼을 클릭해서 확인하면, value 안에 userData라는 프로퍼티가 생성되어있고, 이 안에 username과 email이 추가되어 있는 것을 확인할 수 있다.
    • ngModelGroup은 ngForm, ngModel과 동일하게 ng-dirty, ng-touched, ng-valid 클래스 또한 활용할 수 있고, ngModelGroup이라는 로컬레퍼런스를 추가하여 활용할 수 있다.

15. TD: Form Value 셋팅 및 업데이트하기

이름을 제안해주고, 그 이름을 Username 인풋에 작성되도록 구현해보자. 이를 구현하는 방법은, viewChild의 폼 엘리먼츠를 이용하는 방법과 two-way binding을 활용하는 방법 2가지가 있다. two-way binding을 활용하여 작성되도록 하는 방법은, 앞에서 활용한 방법과 동일하므로, viewChild 폼 엘리먼츠를 이용하는 방법만 작성한다.

  • viewChild 폼 엘리먼츠를 이용하는 방법
export class AppComponent {
    @ViewChild('f') signupForm: NgForm;
    suggestUsername() {
        const suggestedName ="Superuser";
        this.signupForm.setValue({
            userData: {
                username: suggestedName,
                email: '',
            },
            secret: 'pet',
            questionAnswer: '',
            gender: 'male'
        })
    }
}
  • setValue 메서드를 사용하면 우리가 원하는 기능을 동작할 수 있다. 그러나, 이름을 작성하기 전, 이메일 및, 성별 등을 작성한 뒤 이름을 제안해주는 버튼을 클릭하면, 모든 폼의 내용이 다 지워진다는 이슈가 있다. 이는 patchValue 메서드를 사용하여 해결할 수 있다.
  • patchValue 메서드
export class AppComponent {
    @ViewChild('f') signupForm: NgForm;
    suggestUsername() {
        const suggestedName ="Superuser";
        this.signupForm.patchValue({
            userData: {
                username: suggestedName,
            },
        })
    }
}

16. TD: Form Data 사용하기

export class AppComponent {
    user = {
        usernaem: '',
        email: '',
        secretQuestion: '',
        answer: '',
        gender: '',
    }


    onSubmit() {
        this.submitted = true;
        this.user.username = this.signupForm.value.userData.username;
        this.user.email = this.signupForm.value.userData.email;
        this.user.secretQuestion = this.signupForm.value.secret;
        this.user.answer = this.signupForm.value.questionAnswer;
        this.user.gender = this.signupForm.value.gender;
    }
}
<div *ngIf="submitted">
    <h3>Your Data</h3>
    <p>Username: {{user.username}}</p>
    <p>Email: {{user.email}}</p>
    <p>Secret: {{user.secretQuestion}}</p>
    <p>Answer: {{user.answer}}</p>
    <p>Gender: {{user.gender}}</p>
</div>

17. TD: Form 리셋하기

export class AppComponent {
    user = {
        usernaem: '',
        email: '',
        secretQuestion: '',
        answer: '',
        gender: '',
    }


    onSubmit() {
        this.submitted = true;
        this.user.username = this.signupForm.value.userData.username;
        this.user.email = this.signupForm.value.userData.email;
        this.user.secretQuestion = this.signupForm.value.secret;
        this.user.answer = this.signupForm.value.questionAnswer;
        this.user.gender = this.signupForm.value.gender;


        this.signupForm.reset();
    }
}
  • form을 리셋하는 방법은 reset 메서드를 호출해주면된다. 이는 모든 form을 empty 값으로 만드는 것으로, 필요에 따라서 setValue 인자로 보내던 객체를 reset 인자로 보내주면 특정 값으로 리셋시키는 것도 가능하다.
  • 리셋 후에, 클래스 또한 ng-untouched, ng-pristine, ng-invlid로 다시 변경되는 것을 확인할 수 있다.
