02 Angular Complete Info 10

Pipe

1. Pipe란?

<p>{{username | uppercase}}</p>
  • Pipe는 Angular2에 기본적으로 빌트인 되어있는 기능으로, Pipe의 주목적은 템플릿에서 아웃풋 변경이다. 예를 들어 username 이라는 프로퍼티를 화면에 렌더링하고 싶다고 가정했을 때, 우리는 string Interperation을 이용해서 화면을 출력할 것이다. 그런데 username의 데이터 변환없이 화면에 출력할 때만 대문자로 보여졌으면 좋겠다고 생각이 들 때, Pipe를 사용할 수 있다.

2. Using Pipes

<li class="list-group-item"
        *ngFor="let server of servers"
        [ngClass]="getStatusClasses(server)">
    <span class="badge">{{server.status}}</span>
    <strong>{{server.name}}</strong> |
    {{server.instanceType | uppercase}} |
    {{server.started | date}}
</li>
  • uppercase: 아웃풋을 대문자로 변경해준다.
  • date: 아웃풋을 날짜형식으로 변경해준다.

3. Parametrizing Pipes

{{server.started | date: 'fullDate'}}
{{server.started | date: 'fullDate':second paramter}}
  • 파라미터를 넘겨주어 내가 원하는 포맷으로 변경할 수도 있다.
  • 여러개의 파라미터가 존재할 경우, :(colons)을 사용하여 구분할 수 있다.

4. Multiple Pipes로 변경하기

하나의 데이터에 여러개의 pipe를 사용하려면 어떻게할 수 있을까? pipe를 하나씩 오른쪽으로 추가해주면되는데, 일반적으로 파싱되는 순서는 왼쪽에서 오른쪽이다.

{{server.started | date: 'fullDate' | uppercase}}
  • server.started 는 date: ‘fullData’ pipe를 만나 아웃풋이 변형된 뒤, 변형된 아웃풋을 uppercase pipe를 만나 대문자로 변형된다.
  • 여기서 {{server.started | uppercase | date: 'fullDate'}} 이와 같이 두 개의 pipe 순서를 변경하면 에러를 얻게되는데, 에러는 데이터가 스트링이 아니기 때문에 파싱을 못한다는 에러이다. → pipe 순서 중요

5. Custom Pipe 만들기

내가 만들고 싶은 아웃풋이 빌트인되어 있는 pipe에는 존재하지 않는다면, 우리는 pipe를 새로 만들어서 사용할 수도 있다. 현재 만들 pipe는 긴 이름을 가진 서버의 이름을 짧게 만드는 파이프이다.

// shorten.pipe.ts
@Pipe({
    name: 'shorten'
})
export class ShortenPipe implements PipeTransform {
    transform(value: any) {
        if (value.length > 10) {
            return value.splice(0, 10) + ' ...';
        }
        return value;
    }
}
<strong>{{server.name | shorten}}</strong>
  • transform: 변형될 값을 인자로 받는다. 항상 어떤 값을 리턴해야된다. 왜냐하면 pipe는 단지 어떤 값을 넣어서 어떤 값을 얻는 것이기 때문에 새로운 스트링을 리턴해야한다.

6. Parametrizing Custom Pipe

위의 코드에서 10으로 하드코딩 되어있는 부분이 너무 거슬릴 수도 있다. 이를 두 번째 인자를 사용하여 제거할 수도 있다.

// shorten.pipe.ts
@Pipe({
  name: 'shorten'
})
export class ShortenPipe implements PipeTransform {
  transform(value: any, limit: number) {
    if (value.length > limit) {
      return value.splice(0, limit) + ' ...';
    }
    return value;
  }
}
<strong>{{server.name | shorten: 5}}</strong>

7. Example: Filter Pipe

Filter Pipe를 만들어서 Input 값과 동일한 상태값을 가진 서버아이템만 화면에 출력될 수 있도록 하는 요구사항을 충족시켜보자.

<div class="col-xs-12 col-sm-12">
    <input type="text" [(ngModel)]="filteredStatus" />
    <hr>
    <ul class="list-group">
        <li class="list-group=item"
                *ngFor="let server of servers"
                [ngClass]="getStatusClasses(server)">
            <span class="badge">{{server.status}}</span>
            <strong>{{server.name | shorten: 15}}</strong> |
            {{server.instanceType | uppercase }} |
            {{server.started | date: 'fullDate' | uppercase}}
        </li>
    </ul>
</div>
export class AppComponent {
    filteredStatus = '';
}
// filter.pipe.ts
@Pipe({
    name: 'filter'
})
export class FilterPipe implements PipeTransform {
    transform(value: any, filterString: string, propName: string): any {
        if (value.length === 0 || filterString === '') {
            return value;
        }
        // loop
        const resultArray = [];
        for (const item of value) {
            if (item[propName] === filterString) {
                resultArray.push(item);
            }
        }
        return resultArray;
    }
}
<div class="col-xs-12 col-sm-12">
    <input type="text" [(ngModel)]="filteredStatus" />
    <hr>
    <ul class="list-group">
        <li class="list-group=item"
                *ngFor="let server of servers | filter: filteredStatus: 'status'"
                [ngClass]="getStatusClasses(server)">
            <span class="badge">{{server.status}}</span>
            <strong>{{server.name | shorten: 15}}</strong> |
            {{server.instanceType | uppercase }} |
            {{server.started | date: 'fullDate' | uppercase}}
        </li>
    </ul>
</div>

8. Pure & Impure Pipe

하지만 여기에서 한가지 이슈가 있다. 새로 추가된 서버는 필터된 리스트에 추가되지 않는다는 점이다.

<div class="col-xs-12 col-sm-12">
    <input type="text" [(ngModel)]="filteredStatus" />
    <hr>
    <button class="btn btn-primary" (click)="onAddServer()">Add Server</button>
    <ul class="list-group">
        <li class="list-group=item"
                *ngFor="let server of servers | filter: filteredStatus: 'status'"
                [ngClass]="getStatusClasses(server)">
            <span class="badge">{{server.status}}</span>
            <strong>{{server.name | shorten: 15}}</strong> |
            {{server.instanceType | uppercase }} |
            {{server.started | date: 'fullDate' | uppercase}}
        </li>
    </ul>
</div>
export class AppComponent {
    onAddServer() {
        this.server.push({
            instanceType: 'small',
            name: 'New Server',
            status: 'stable',
            started: new Date(15, 1, 2022),
        });
    }
}
  • 이슈 동작방법: Add Server 버튼 클릭(status: stable) > 필터 입력창에 stable 작성 > 새로만들어진 Server는 리스트에 뿌려지지 않음
  • 정상 동작방법: 필터 입력창에 stable 작성 > Add Server 버튼 클릭(status: stable) > 필터 입력창에 빈값 작성 후 다시 stable 작성 > 새로만들어진 Server가 리스트에 뿌려진다.
  • 그렇다면 이렇게 이슈가 생기는 이유는 무엇일까? 사실 이 것은 이슈가 아닌 정상동작이다. Angular는 데이터(server 배열)가 변경될 때마다 데이터에 대한 pipe를 다시 실행하지 않기 때문이다. 그러나 입력을 추가하거나 파이프의 입력을 변경하면 재계산이 트리거되고 실제로 파이프가 데이터에 다시 적용되도록 트리거된다. 그렇기 때문에 필터 입력창에 빈값 작성 후 다시 stable을 작성하는 동작을 통해 우리는 새로만들어진 Server를 리스트에 호출할 수 있게 되는 것이다.
  • 이는 좋은 퍼포먼스를 위해 꼭 필요한 동작으로, 만약 데이터가 변경될 때마다 재계산이 되도록 구현되어 있다면, Angular는 페이지의 데이터가 변경될 때마다 pipe를 재실행 하게되므로 성능에 큰 비용을 쓰게 된다.
  • 그런데도 불구하고, 꼭 사용해야하는 기능이라고 하면, pipe 데코레이터에 pure 프로퍼티를 추가함으로써 이를 사용할 수 있다.
@Pipe({
    name: 'filter',
    pure: false
})
  • {pure: false}를 추가하면 Filter Pipe는 정상적으로 동작하게 된다(데이터변경에도 감지 가능한 pipe가 된다). 왜냐하면 파이프의 데이터가 변경될때마다 다시 계산을 하기 때문이다.
  • 이제는 페이지에서 변경사항이 있을때마다 다시 계산하도록 구현이 되어있다. 이는 큰 비용을 감수해야하며, 이에 대해 잘 알고 있지만 그래도 꼭! 필요하다고 생각할 때, 그 때 사용하는 것이 낫다. → 무분별하게 사용하지 말자

9. async Pipe

HTTP Call로부터 리턴되는 데이터를 변형할 필요가 있다면, 어떻게 pipe를 사용할 수가 있을까?

export class AppComponent {
    appStatus = new Promise((resovle, reject) => {
        setTimeout(() => {
            resolve('stable');
        }, 2000);
    }); // offline, critical...
}
<div class="col-xs-12 col-sm-12">
    <input type="text" [(ngModel)]="filteredStatus" />
    <hr>
    <button class="btn btn-primary" (click)="onAddServer()">Add Server</button>
    <br><br>
    <h2>App Status: {{appStatus | async}}</h2>
    <ul class="list-group">
        <li class="list-group=item"
                *ngFor="let server of servers | filter: filteredStatus: 'status'"
                [ngClass]="getStatusClasses(server)">
            <span class="badge">{{server.status}}</span>
            <strong>{{server.name | shorten: 15}}</strong> |
            {{server.instanceType | uppercase }} |
            {{server.started | date: 'fullDate' | uppercase}}
        </li>
    </ul>
</div>
  • 여기서 async 속성을 사용하지 않는다면, appStatus는 [object object]로 출력된다. 이는 Promise가 객체이므로 정상동작을 하고 있는 것이다. 하지만 우리는 2초 후에 들어오는 데이터를 출력하고 싶다.
  • async 속성을 사용하여 2초 후에 들어오는 데이터를 출력할 수 있다. async의 동작원리는 Promise임을 인식하고 부수적으로 appStatus를 구독하는 것이라고 생각할 수 있다. 즉, 자동으로 구독하고 2초 뒤에 들어온 데이터를 화면에 뿌려주는 것과 같은 원리이다.
