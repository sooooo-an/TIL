04 Angular Complete Info 03

3. Directive

  1. Attribute vs Structural Directive

  • Attribute Directive나 Structural Directive는 기본적으로 같은 역할을 하지만, 둘러싼 DOM의 구조를 변경하는지에 대한 유무에 따라 두 Directive로 나뉜다.

    • 예를 들어, ngIf는 조건이 false일 때, 엘리먼트가 DOM에서 삭제되므로 뷰 컨테이너 전체가 영향을 받지만, Attribute Directive에서는 엘리먼트를 삭제하지 않고, backgroundColor 등 해당 엘리먼트의 속성만 변경한다.

  • Attribute Directive

    • 일반 HTML Attribute처럼 보인다. (데이터 바인딩 또는 이벤트 바인딩 사용 가능)
    • 추가된 엘리먼츠에만 영향을 미치거나 변경된다.

  • Structural Directive

    • 일반 HTML Attribute처럼 보이나, 앞에 *가 존재한다.
    • DOM의 전체 영역에 영향을 준다. (엘리먼츠가 추가되거나 삭제되거나)

  1. Attribute Directive 생성

      import {} from '@angular/core';
    

      @Directive({
        selector: 'appBasicHighlight'
      })
      export class BasicHighlightDirective implements OnInit {
        constructor(private elementRef: ElementRef) {}
    

        ngOnInit() {
            this.elementRef.nativeElement.style.backgroundColor = 'green';
        }
      }
  • 사용방법: <div [appBasicHighlight]></div> → Attribute Directive 이기 때문에 대괄호를 사용한다.
  • appBasicHighlight: 각 엘리먼츠를 선택한다는 뜻이다. → 셀렉터에 이렇게 작성한 경우, Attribute Directive인 경우 각 엘리먼츠에 대괄호를 추가해주어야 한다.
  • [appBasicHighlight]: Attribute 스타일. 셀렉터에 대괄호를 추가해서 작성하게 되면, 각 엘리먼트에 appBasicHighlight 라고 작성하더라도, Attribute Directive로 인식된다는 것을 의미한다.
  • 엘리먼츠에 클래스 및 스타일이 존재 > Attribute Directive와 겹치는 스타일 > Attribute Directive가 엘리먼츠의 클래스 및 스타일을 덮는다.
  1. Attribute Directive 보다 더 나은 Renderer를 이용

  • Attribute Directive는 스타일을 변경하는데 최고의 방법은 아니다. → 엘리먼츠에 직접 액세스하는 것은 좋은 방법이 아니기 때문에

  • Angular는 실제로 DOM없이 템플릿을 렌더링할 수도 있고, 이러한 프로퍼티를 사용하지 못할 수도 있기 때문에 다른 도구를 사용해야한다 → Renderer를 사용하기

  • Renderer Example

      import { Directive, Renderer2, ElementRef } from '@angular/core';
    

      @Directive({
        selector: '[appBetterHighlight]'
      })
      export class BetterHighlightDirective implements OnInit {
        constructor(
          private elRef: ElementRef,
          private renderer: Renderer2
        ){
        }
    

        ngOnInit() {
          this.renderer.setStyle(this.elRef.nativateElement, 'background-color', 'blue', false, false);
        }
      }

  • 첫번째 인자: 스타일을 설정할 엘리먼츠로, 스타일을 추가하기 위해서는 native element가 필요한데, 참조 자체를 넘길 수가 없으므로 this.elRef.nativateElement 로 추가해주어야 한다.

  • 네번째 인자: 몇 가지 플래그를 설정할 수 있으며 이는 옵션 사항이다. 여기서 설정할 수 있는 것은 중요한 태그를 추가하려는 경우와 같이 느낌표, 다른 스타일을 재정의 하는 스타일에 대한 재정의 등이다.

  • Renderer가 DOM에 접근하는 것이 더 나은 이유는 무엇일까?

    • Angular는 서비스 워커와도 작동하고, DOM에 액세스할 수 없는 환경인 경우에도 브라우저에서 실행하는 것에 국한되지 않는다. 따라서 네이티브 엘리먼츠와 이 엘리먼츠의 스타일에 직접 액세스하여 DOM을 변경하려고 하면 상황에 따라 오류가 발생할 수 있다. (그러나 이런 오류는 흔하지 않다.)

  1. HostListener → 호스트 이벤트 호출

      import { Directive, Renderer2, ElementRef, HostListener } from '@angular/core';
    

      @Directive({
        selector: '[appBetterHighlight]'
      })
      export class BetterHighlightDirective implements OnInit {
        @HostListener('mouseenter') mouserover(eventData: Event) {
            this.renderer.setStyle(this.elRef.nativateElement, 'background-color', 'blue', false, false);
        }
        @HostListener('mouseleave') mouserleave(eventData: Event) {
            this.renderer.setStyle(this.elRef.nativeElement, 'background-color', 'transparent', false, false);
        }
    

        constructor(
            private elRef: ElementRef,
            private renderer: Renderer2
        ) {
        }
    

        ngOnInit() {
        }
      }
  • @HostListener는 어떤 이벤트가 발생할 때마다 트리거 될 수 있으며, 해당 이벤트는 @HostLinstener의 인자로 지정된다. @HostLinstener(이벤트) → 여기서 이벤트는 DOM 엘리먼츠에서 지원하는 이벤트 모두 사용 가능하다.
  • 한계: @HostLinstener는 엘리먼츠의 이벤트를 수신하는데 편리한 방법일 뿐이다. 그러나 우리는 mouseenter 이벤트 뿐만 아니라 필요하지 않은 이벤트 데이터 또한 얻게 된다.
  1. HostBinding → 호스트 프로퍼티 바인딩

  • 다른 데코레이터를 사용하는 방법 → renderer를 사용하는게 나쁜 방법은 아니지만, 스타일을 변경하는 더 좋은 방법

  • HostBinding

        import { Directive, Renderer2, ElementRef, HostListener, HostBinding } from '';
    

        @Directive({
          selector: '[appBetterHighlight]'
        })
        export class BetterHighligtDirective implements OnInit {
          @HostBinding('style.backgroundColor') backgroundColor = 'transparent';
          @HostListener('mouseenter') mouseover(eventData: Event) {
            this.backgroundColor = 'blue';
          }
    

          @HostListener('mouseleave') mouseleave(eventData: Event) {
            this.backgroundColor = 'transparent';
          }
        }

  • @HostBinding('style.backgroundColor'): @HostBinding의 인자로는 속성을 정의하는 문자열을 전달한다.

  • @HostBinding('style.backgroundColor') backgroundColor = 'transparent';: 주의할 점: 마우스를 가져가기 전에 오류가 발생하지 않도록 초기값 설정 필요

  1. Directive Properties 바인딩

  • 위의 예시는 동적으로 스타일을 추가할 수 없다. 이 Directive를 재사용할 수 있도록 만들거나, 다른 패키지와 함께 사용하도록 만들기 위해서는 값을 동적으로 설정할 수 있어야 한다.

  • 동적 바인딩 Directive Example

        import { Directive, Renderer2, ElementRef, HostListener, HostBinding } from '';
    

        @Directive({
          selector: '[appBetterHighlight]'
        })
        export class BetterHighligtDirective implements OnInit {
          @Input() defaultColor = 'transparent';
          @Input() highlightColor = 'blue';
          @HostBinding('style.backgroundColor') backgroundColor = this.defaultColor;
          @HostListener('mouseenter') mouseover(eventData: Event) {
            this.backgroundColor = this.highlightColor;
          }
    

          @HostListener('mouseleave') mouseleave(eventData: Event) {
            this.backgroundColor = this.defaultColor;
          }
    

          ngOnInit() {
            this.backgroundColor = this.defaultColor;
          }
        }

  • backgroundColor 초기화를 프로퍼티에서 하게되면, 처음 렌더링될 때, defaultColor가 적용되지 않는 이슈가 있다. → ngOnInit에 넣어야됨 (렌더되기 전, value가 존재한 후)

  • 우리가 템플릿에서 value([defaultColor]=”’yellow’”)를 넘기는데, Angular는 default color가 Directive의 프로퍼티에 바인딩 된지 안된지 어떻게 알아?

    ⇒ Angular는 엘리먼츠의 기본 프로퍼티에 도달하기 전에 먼저 엘리먼츠의 Directive를 확인하기 때문에 알 수 있다

  • 즉, Directive의 프로퍼티를 대괄호를 이용하여 동일한 엘리먼츠에 간단하게 배치하여 해당 프로퍼티에 바인딩할 수 있다. ([defaultColor]=”’yellow’”)

  • ngClass 또한 대괄호를 이용한 바인딩이 가능하다.

  • 입력되는 인자의 값과 Directive의 이름을 동일하게 하여 사용하는 방법 → Directive에 대괄호를 사용하여 함께 이용가능.

        import { Directive, Renderer2, ElementRef, HostListener, HostBinding } from '';
    

        @Directive({
          selector: '[appBetterHighlight]'
        })
        export class BetterHighligtDirective implements OnInit {
          @Input() defaultColor = 'transparent';
          @Input('appBetterHighlight') highlightColor = 'blue';
          @HostBinding('style.backgroundColor') backgroundColor = this.defaultColor;
          @HostListener('mouseenter') mouseover(eventData: Event) {
            this.backgroundColor = this.highlightColor;
          }
    

          @HostListener('mouseleave') mouseleave(eventData: Event) {
            this.backgroundColor = this.defaultColor;
          }
    

          ngOnInit() {
            this.backgroundColor = this.defaultColor;
          }
        }
      <p [defaultColor]="'yellow'" [appBetterHighlight]="'red'"></p>
  1. Structural Directive에서 일어나는 일

  • *ngIf, *ngFor: *의 표시는 Angular에게 Structural Directive라는 것을 알려주는 표시이다. ([ngIf] 로도 사용가능하다)

        <ng-template [ngIf]="!onlyOdd">
          <div></div>
        </ng-template>

  • ng-template: ng-template 자체는 렌더링 되지 않지만, Angular가 이 템플릿이 true로 판단될 때, 사용할 템플릿을 정의할 수 있기 때문에 ng-template으로 감싸져 있다.

  • ng-template을 사용할 때는 *와 함께 사용하지 않는다. → 이 패턴은 *이 변환된 형태이기 때문이다. (*ngIf === <ng-template [ngIf]=”true”>)

  1. Structural Directive 생성

      import { Directive, Input, TemplateRef, ViewContainerRef } from '@angular/core';
    

      @Directive({
        selector: '[appUnless]'
      })
    

      export class UnlessDirective {
        @Input() set unless(condition: boolean) {
            if (!condition) {
                this.vcRef.createEmbeddedView(this.templateRef);
            } else {
                this.vcRef.clear();
            }
        }
    

        constructor(
            private templateRef: TemplateRef<any>,
            private veRef: ViewContainerRef,
        ) {}
      }

  • set unless는 unless가 외부에서 변경될 때마다 변경된다. 즉, condition이 변경될 때마다 이 condition의 일부 파라미터가 변경된다.

  • Angular는 어떻게 condition을 파악하고, 어떻게 사용하고, 어떻게 표시할 수 있을까?

    • Angular는 *을 사용하여 변환시키기 때문에 appUnless directive는 ng-template 컴포넌트에 배치되어 있을 것이다.
    • 따라서 이 템플릿에 액세스할 수 있고, 문서에서 렌더링하는 위치에 액세스할 수 있어야 한다. 즉, 둘 다 주입할 수 있어야 한다.
    • 템플릿 주입: templateRef를 추가하여 주입할 수 있다.
    • viewContainer 주입: ViewContainerRef를 추가하여 주입할 수 있다.

  • createEmbeddedView: view 내부에 viewContainer 안에 this.templateRef를 추가한다. this.vcRef.createEmbeddedView(this.templateRef)은 정확히 거기에서 만든 템플릿에 대한 참조이고, 우리가 필요한 것이다.

  • HTML 코드 예시 → 에러 구문

        <div *appUnless="onlyOdd"></div>

  • *은 ng-template 구문으로 자동으로 변환한 다음, appUnless인 Directive 이름에 프로퍼티 바인딩을 시도한다. 따라서 여기서 프로퍼티가 appUnless인 Directive 이름을 공유하는지 확인해야한다.

  • 이를 에러가 나지 않게 하기 위해서는 아래의 구문으로 수정하면된다.

      import { Directive, Input, TemplateRef, ViewContainerRef } from '@angular/core';
    

      @Directive({
        selector: '[appUnless]'
      })
    

      export class UnlessDirective {
        @Input() set appUnless(condition: boolean) {
          if (!condition) {
            this.vcRef.createEmbeddedView(this.templateRef);
          } else {
            this.vcRef.clear();
          }
        }
    

        constructor(
          private templateRef: TemplateRef<any>,
          private veRef: ViewContainerRef,
        ) {}
      }

  1. ngSwitch 이해하기

      <div [ngSwitch]="value">
        <p *ngSwitchCase="5">Value is 5</p>
        <p *ngSwitchCase="10">Value is 10</p>
        <p *ngSwitchCase="100">Value is 100</p>
        <p *ngSwitchDefault>Value is Default</p>
      </div>
  • 프로퍼티 바인딩을 사용하여 ngSwitch에 바인드되므로 이 시점에서는 *없이 값에 바인드한다.
  • *ngSwitchCase: *은 Angular에 의해 변형된다.
