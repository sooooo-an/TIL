09 Xss

XSS - Cross-site scripting

1. XSS(Cross-site-scripting)이란?

  • 웹 사이트에 악성 클라이언트 사이드 코드를 주입할 수 있도록 하는 보안 공격으로 주입된 코드는 피해자에 의해 실행되며, 공격자는 엑세스 제어를 무시하고 사용자로 가장할 수 있다.
  • XSS은 웹앱이 충분한 유효성 검사 또는 인코딩을 사용하지 않을 경우에 성공하게 된다. 브라우저는 악성 스크립트가 신뢰할 수 없는 것임을 감지할 수 없으므로, 사용자로 가장한 공격자에게 쿠키, 세션 토큰 또는 기타 중요한 사이트별 정보에 대한 엑세스를 제공하거나 악성 스크립트가 HTML 내용을 다시 쓸 수 있도록 허용한다.

2. XSS(Cross-site-scripting)가 일어나는 시점

  1. 데이터가 신뢰할 수 없는 소스(Web request)를 통해 웹앱에 들어갈 때 발생
  2. 동적 콘텐츠가 악성 콘텐츠에 대해 검증되지 않고, 웹 사용자에게 전송될 때 발생

3. XSS(Cross-site-scripting)가 포함되는 코드의 범위

  • 주로 JavaScript 코드에 포함되지만 HTML 코드나 Flash 및 실행되는 브라우저의 다른 코드에 포함될 수도 있다.
  • XSS 공격 방법
    • XSS 공격 범위는 제한이 없지만, 보통 쿠키나 세션 정보같은 프라이빗 데이터를 공격자에게 전송
    • 공격자에 의해 제어되는 웹페이지로 공격 대상을 리디렉션
    • 취약한 사이트를 가장하여 사용자의 컴퓨터에서 다른 악의적인 작업을 수행

4. XSS 공격 범주

  1. Stored XSS Attacks
    • 주입된 스크립트가 타겟 서버에 영구적으로 저장되어, 공격을 당한 타겟 서버는 브라우저가 데이터 요청을 보낼 때, 서버에서 이 악성 스크립트를 검색한다.
  2. Reflected XSS Attacks
    • 사용자가 속임수로 악성 코드를 클릭하거나 특수하게 조장된 양식을 제출하거나 악의적인 사이트를 탐색하면 주입도니 코드가 취약한 웹 사이트로 이동한다. 그 웹서버는 오류 메시지, 검색 결과 또는 요청의 일부로 서버로 전송된 데이터를 포함하는 기타 응답과 같이 주입된 스크립트를 다시 사용자의 브라우저에 반영한다. 브라우저는 사용자가 이미 상호 작용한 “신뢰할 수 있는” 서버에서 응답한 것으로 가정하기 때문에 코드를 실행한다.
  3. DOM-based XSS Attacks
    • payload는 피해자의 브라우저에서 오리지널 클라이언트 사이드 스크립트가 사용하는 DOM 환경을 변경된 결과로 실행한다. 즉, 페이지 자체는 변경되지 않지만 DOM 환경에 대한 악의적인 수정으로 인해 페이지에 포함된 클라이언트 사이드 스크립트가 기대한 바와 다르게 실행된다.

5. XSS 해결책

  • 기존의 해결책
    • 클라이언트에서 백엔드로 데이터 전송 시 HTML 태그를 Entity 태그로 변경 ( < ⇒ &lt; )
    • 백엔드에서 HTML 태그를 받았을 때, Entity 태그로 변경하여 저장
    • 프론트에서 화면에 출력할 때, Entity 태그를 HTML 태그로 변환해서 출력
  • 변경된 해결책
    • 클라이언트에서 백엔드로 데이터 전송 시 HTML 태그 그대로 저장
    • 프론트에서 화면에 출력할 때, HTML 태그를 Entity 태그로 변환 후, 렌더링 과정에서 Entity 태그를 HTML 태그로 치환 ( < ⇒ &lt; ⇒ < )
      // 1. get data from server 
      // server data
      const info = {
          name: '<script>alert('XSS')</script>',
          id: '12312312',
          //....
      };
      

      // 2. convert entity tag from html tag 
      // using having library or util
      const convertInfo = {
          ...info, 
          name: escapeHtmlString(info.name), //&lt;script&gt;alert('XSS')&lt;/script&gt;
      };
      

      // 3. rendering 
      <p [innerHTML]="convertInfo.name"></p>
      <p [innerHTML]="(info.name | escaptHtmlStringPipe)"></p>
