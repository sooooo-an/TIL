Forms2 - Reactive Form

1. Reactive: Reactive Form 셋업하기

export class AppComponent {
  signupForm: FormGroup;
}
  • formGroup은 초기값 셋팅을 할 필요가 없다. 그러나 Reactive From이 정상동작하려면, app.module.ts에 FormModule(Template-Driven으로 작성 시 추가해야하는 모듈)이 아닌, ReactiveFormsModule을 추가해주어야 한다. ReactiveFormsModules은 폼을 스스로 만들고, 그 폼을 우리의 HTML코드에 동기화시키기 위해 필요한 모든 도구들이 들어있는 모듈이다.

2. Reactive: Creating Form in Code

export class AppComponent implements OnInit {
    signupForm: FormGroup;


    ngOnInit() {
        this.signupForm = new FormGroup({
            'username': new FormControl(null),
            'email': new FormControl(null),
            'gender': new FormControl('female'),
        });
    }
}
  • 프로퍼티 선언부에 즉시 값을 셋팅해도 되지만, ngOnInit 라이프사이클에서 초기화 작업을 통해 렌더링작업을 수행하므로 ngOnInit에서 셋팅을 하는게 더 나은 방법이다. → 왜 더 나은 방법인지 코드작성필요
  • FormGroup: 클래스로 새로운 인스턴스를 추가할 수 있다. 인자로는 객체를 받는데, 이 객체에 폼 그룹에 포함되는 각 프로퍼티 및 프로퍼티 초기값을 추가할 수 있다.
  • FormControl
    • 첫 번째 인자: 초기값을 넣어준다.
    • 두 번째 인자: validators의 배열을 넣어준다.
    • 세 번째 인자: 잠재적인 asynchronous validator을 넣어준다.

3. Reactive: Form과 HTML코드 동기화시키기

<form [formGroup]="signupForm">
    <div class="form-group">
        <label for="username">Username</label>
        <input
            type="text"
            id="username"
            formControlName="usernmae"
            class="form-control"/>
    </div>
    <div class="form-group">
        <label for="email">Email</label>
        <input
            type="text"
            id="email"
            formControlName="email"
            class="form-control"/>
    </div>
    <div class="radio" *ngFor="let gender of genders">
        <label for="username">
            <input
                type="radio"
                formControlName="gender"
                [value]="gender"/>{{gender}}
        </label>
    </div>
</form>
  • [formGroup]: formGroup 프로퍼티 디렉티브를 추가하게되면, Angular에게 이 form에 프로퍼티 바인딩을 설정할 것임을 알려준다.
    • signupForm을 값으로 설정해줌으로써, 이 폼을 항상 동기화시킬 수 있게 되었다.
  • formControlName: 프로퍼티 바인딩을 이용할 수도 있고, string으로 사용할 수도 있다. 이 작업을 통해 폼 안의 프로퍼티와 HTML 엘리먼츠를 동기화시킬 수 있다.

4. Reactive: Submitting the Form

<form (ngSubmit)="onSubmit()"></form>
export class AppComponent implements OnInit {
    onSubmit() {
        console.log(this.signupForm);
    }
}
  • template driven에서 사용했던 방식과 동일하게 ngSubmit 을 사용하면 된다.
  • template driven과 다른점
    • 폼에 로컬레퍼런스(#f=”ngForm”)를 할 필요가 없다. 스스로 만들었기 때문에 타입스크립트 코드에 접근할 수 있기 때문이다. → 모델, 애플리케이션 모델에 바인드하여 폼 구조가 모델의 구조와 일치하는지 쉽게 확인 가능하다.

5. Reactive: Adding Validation

template driven 폼에서 사용했던 required 같은 내장 디렉티브를 Reactive 폼에서는 사용할 수 없다. 왜냐하면 템플릿에서 폼을 구성한 것(ngModel 디렉티브 추가)이 아니라, 직접 폼을 만들어서 formControlName, formGroup 디렉티브를 동기화만 시켰기 때문이다.

export class AppComponent implements OnInit {
    signupForm: FormGroup;


    ngOnInit() {
        this.signupForm = new FormGroup({
            'username': new FormControl(null, Validator.reuqired),
            'email': new FormControl(null, [Validator.reuqired, Validator.email]),
            'gender': new FormControl('female'),
        });
    }
}
  • Validator.reuqired를 호출하는 것이 아닌, 정적 메서드만 보낸다 → 이 메서드에 대한 참조(메모리 주소)만 전달한다. Angular는 이 FormControl의 입력이 변경되었음을 감지할 때마다 메서드를 실행한다.
  • 배열을 사용하면, 여러개의 유효성 검사를 할 수 있다.

6. Reactive: Getting Access to Controls

template driven 폼에서는 input에 #username=”ngModel” 이라는 로컬레퍼런스를 추가해주어야지 데이터에 접근이 가능했다.

<div class="form-group">
  <label for="username">Username</label>
  <input
    type="text"
    id="username"
    formControlName="username"
    class="form-control"
  />
  <span
    *ngIf="!signupForm.get('username').valid && signupForm.get('username').touched"
    class="help-block">Please enter a valid username!</span>
</div>


<span
  *ngIf="!signupForm.valid && signupForm.touched"
></span>
  • Reactive 폼에서는 signupForm의 get 메서드를 이용하여 더욱 수월하게 접근할 수 있다. (전체 폼의 valid 프로퍼티에도 접근 가능)

8. Reactive: Arrays of Form Controls (FormArray)

동적으로 formControl 추가하기 → onAddHobby 메서드가 호출되면 hobbies의 배열에 새로운 hobby를 추가해보자

<h4>Your Hobbies</h4>
<button
    class="btn btn-primary"
    type="button"
    (click)="onAddHobby()">Add Hobby</button>
export class AppComponent implements OnInit {
    ngOnInit() {
        this.signupForm = new FormGroup({
            'userData': new FormGroup({
                'username': new FormControl(null, Validator.required),
                'email': new FormControl(null, [Validators.reuqired, Validators.email]),
            }),
            'gender': new FormControl('female'),
            'hobbies': new FormArray([]),
        })
    }
onAddHobby() {
    const control = new FormControl(null, Validators.required);
    (<FormArray>this.signupForm.get('hobbies')).push(control)
}

}

- 유저가 onAddHobby를 호출하면, 취미는 항상 default value가 없이 생성된다. 이제 이 코드와 템플릿 코드를 동기화시켜야한다.  → formArrayName 디렉티브 추가하기
```html
<div formArrayName="hobbies">
    <h4>Your Hobbies</h4>
    <button
        class="btn btn-primary"
        type="button"
        (click)="onAddHobby()">Add Hobby</button>


    <div class="form-group"
             *ngFor="let hobbyControl of signupForm.get('hobbies').controls;

9. Reactive: Creating Custom Validators

Reactive 폼은 Template Driven과 다르게 커스텀 유효성검사 또한 가능하다. 외국인 유저 이름에만 해당되는 Validator를 만들어보자

export class AppComponent implements OnInit {
    forbiddenUsernames = ['Chris', 'Anna'];
ngOnInit() {
    this.signupForm = new FormGroup({
        'userData': new FormGroup({
            'username': new FormControl(null, [Validators.required, this.forbiddenNames.bind(this)]),
            'email': new FormControl(null, [Validators.requried, Validators.email]),
        }),
        'gender': new FormControl('female'),
        'hobbies': new FormArray([])
    });
}


forbiddenNames(control: FormControl): {[s: string]: boolean} {
    if (this.forbiddenUsernames.indexOf(control.value) !== -1) {
        return {'nameIsForbidden': true}
    }
    return null;
}

}

- Validator는 FormControl의 validity를 체크해주기 위해서 Angular가 자동으로 실행시켜주는 함수로 인자를 받아 Angular가 리턴값을 올바르게 처리할 수 있도록 리턴시킨다.
  - 왜 this를 바인딩해줄까?
    - forbiddenNames 메서드를 확인하면, forbiddenUsernames에 접근하고 있는데, AppComponent 클래스에서는 forbiddenUsernames가 존재하기 때문에 여기에 접근하는데는 문제가 없다. (AppComponent에서의 this는 클래스 자기 자신이다)
    - 그러나 forbiddenNames를 호출하고 있는 주체는 지금 AppComponent가 아닌 Angular이다. 우리는 validator 배열 안의 this.forbiddenNames 이라는 메서드를 AppComponent에서 호출하는 것이 아니라 참조값만 보내주고 있기 때문에 결국, Angular가 이 메서드를 호출시켜준다는 건데, Angular에는 forbiddenUsernames 이라는 배열이 없으므로 에러를 뱉어내게 된다. → bind를 이용하여 해결 가능
  - forbiddenNames 메서드를 보면, control.value가 forbiddenUsernames배열에 존재한다면 외국인이라는 뜻이므로 true를 리턴한다. 그리고 존재하지 않다면 null을 리턴하면 된다.
    - invalid: 오류코드(nameIsForbidden)와 함께 객체를 리턴한다.
    - valid: null or 빈값을 리턴한다  {’nameIsForbidden’: false} (절대 false를 리턴해서는 안된다)


### 10. Reactive: Using Error Codes
> 전체 폼의 error 프로퍼티를 확인하면 null임을 확인할 수 있다. 그러나 controls 객체의 username 안의 error 프로퍼티를 확인하면 {nameIsForbidden: true} 이라는 에러코드를 확인할 수 있다. 이는 Angular가 오류를 개별 컨트롤에 추가하기 때문이다.
> 우리는 이 에러메시지를 활용하여 username이 빈값이면 required 라는 에러메시지를, nameIsForbidden 이라면 invalid username 이라는 에러메시지를 스위칭해보자.
```html
<span *ngIf="!signupForm.get('userData.username').valid && signupForm.get('userData.username').touched"
          class="help-block">
    <span *ngIf="signupForm.get('userData.username').errors['nameIsForbidden']">
        This name is invalid!
    </span>
    <span *ngIf="signupForm.get('userData.username').errors['required']">
        Please field is required!
    </span>
</span>

11. Reactive: Creating a Custom Async Validator

폼의 유효성을 프론트단에서만 체크하는 폼도 있지만, 웹서버까지 도달을 해야만 알 수 있는 폼도 있다. 이 동작은 비동기로 동작하므로 응답이 즉시 돌아오지는 않는다. 그렇기 때문에 우리는 비동기 validator도 필요하다. → asynchronous validator 만들기

export class AppComponent implements OnInit {
    ngOnInit() {
        this.signupForm = new FormGroup({
            'userData': new FormGroup({
                'username': new FormControl(null, [Validators.required, this.forbiddenNames.bind(this)]),
                'email': new FormControl(null, [Validators.required, Validators.email], this.forbiddenEmails),
            }),
            'gender': new FormControl('female'),
            'hobbies': new FormArray([])
        })
    }
forbiddenEmails(control: FormControl): Promise<any> | Observable<any> {
    const promise = new Promise<any>((resolve, reject) => {
        setTimeout(() => {
            if (control.value === 'test@test.com') {
                // invalid
                resolve({'emailIsForbidden': true});
            } else {
                // valid
                resolve(null);
            }
        }, 1500);


        return promise;
    });
}

}

- 에러코드를 boolean 값으로 해서 리턴하는 것이 아닌, 이 리턴값을 Promise나 Observable 안에 맵핑하여 리턴해주어야 한다.
- FormControl의 두 번째 인자인 validator array에 추가하는 것이 아닌, 세 번째 인자인 비동기 validator에 추가해야한다. 여기서도 두 번째 인자와 동일하게 레퍼런스만 넘겨주면된다.
- 버튼을 누르고 엘리먼츠를 확인해보면, ng-pending이라는 클래스가 추가된 것을 확인할 수 있다. 그리고 1500ms 이후에 이 클래스가 제거되고, ng-valid 라는 클래스가 추가된 것을 확인할 수 있다.


### 12. Reactive: Reacting to Status or Value Changes
```typescript
export class AppComponent implements OnInit {
    ngOnInit() {
        this.signupForm.valueChanges.subscribe((value) => {
            console.log(value);
        });


        this.signupForm.statusChange.subscribe((status) => {
            console.log(status);
        });
    }
}
  • valueChange: 폼이 변경될 때마다 호출된다. 여기서 전체의 폼이 아닌 개별의 폼 컨트롤을 부를 수도 있다.
  • statusChange: 폼에 대한 validator를 체크할 수 있다. INVALID 인지 VALID 인지에 대한 값을 확인할 수 있다. 만약 비동기 validator로 추가하게 된다면, statusChanges의 상태가 PENDING이 되었다가 VALID로 변경되는 것을 확인할 수 있다.

13. Reactive: Setting and Patching Values

Template Driven 폼과 동일하게 Reactive 폼도 setValue, patchValue 및 reset을 이용할 수 있다.

export class AppComponent implements OnInit {
    ngOnInit() {
        this.signupForm.setValue({
            'userData': {
                'username': 'Max',
                'email': 'max@test.com'
            },
            'gender': 'male',
            'hobbies': []
        })
    this.signupForm.patchValue({
        'userData': {
            'username': 'Anna'
        },
    })


    this.signupForm.reset();
}

}
